# MVP範囲

## MVP定義
### 目的
- 「人のプランをコピーしてGoogle Mapで使う」という体験の価値検証。
- 「ムービーがもらえるなら面倒な振り返り入力をするか」という仮説検証。

### 対象ユーザー
- 初期は自分たち（ファウンダー）と、身近な旅行好きパパママ友人数名。
- その後、SNSで募集したβテスター。

## スコープ
### MVP に含める ✅
- ユーザー登録・プロフィール（子供年齢）
- 旅程の作成・編集（基本機能）
- 旅程の検索・閲覧（一覧表示）
- GoogleマイマップへのKMLエクスポート
- 振り返り入力（評価・コメント・失敗タグ）
- BGM・アニメーション付きのムービー生成
- 子供向け「旅のしおり」PDF生成機能

### MVP に含めない ❌
- ユーザー間のフォロー・DM機能
- 高度な動画編集機能（音楽選択など）
- ホテル・航空券の予約機能
- アプリ版（まずはWebのみ）

## リスクと対策
| リスク | 対策 |
|--------|------|
| Google Map APIの制限/コスト | 初期はAPIを使わず、KMLファイル出力を活用する案も検討。 |
| 初期コンテンツ不足 | サービス開始前に、自分たちや友人の過去の旅行データを入力して「見本」を数十件用意しておく。 |
