# UI/UX方針

## 1. 全体の方向性
### デザインテーマ：World Traveler（空港・パスポート・旅）
- ユーザーがサイトを開いた瞬間から「旅が始まる」ワクワク感を感じさせるデザイン。
- 国内外対応を意識し、特定の国に偏らないユニバーサルかつモダンな「空港」のイメージ。

### キーワード
- **Excitement (ワクワク)**: 出発ボード、チケット、スタンプ。
- **Trust (信頼)**: 失敗情報もしっかり伝える誠実さ。情報の見やすさ。
- **Memory (思い出)**: アルバムのような温かみ。

## 2. ビジュアルモチーフ
- **トップページ/検索**: 空港の「出発案内板（Departure Board）」風のリスト表示。パタパタと切り替わるアニメーション演出など。
- **プラン詳細**: 「航空券（Boarding Pass）」のようなチケット型デザイン。ミシン目やバーコードのあしらい。
- **マイページ**: 「パスポート」風。ユーザーランクが上がると、スタンプが増えていく。背景にうっすらと世界地図の透かし。
- **失敗アラート**: 空港の「Warning」「Caution」サインのような、視認性が高いがデザインに馴染むアイコン（黄色と黒の警戒色など）。

## 3. カラー方針
- **メインカラー**: 深いネイビー（パスポートの色、夜空） ＋ ゴールド（高級感、スタンプ）。
- **アクセント**: 
    - 成功・おすすめ：鮮やかなスカイブルー（空）。
    - 失敗・注意：アラートイエロー（注意喚起）。
- **ベース**: ホワイト〜ライトグレー（清潔感）。

## 4. UX原則
### 重視する体験
1. **直感的なリスク回避**: タイムラインを眺めるだけで「あ、ここは注意だな」と直感的にわかるアイコン設計。
2. **入力の楽しさ**: 面倒な入力作業を、「スタンプを押す」「旅の記録を残す」というポジティブなアクションに変える演出。
3. **Map連携のシームレスさ**: ワンタップでGoogle Mapが立ち上がり、すぐにナビが開始できること。

### 避けたいUX
1. **情報の羅列**: 文字だらけで読む気をなくす画面（アイコンや写真を多用する）。
2. **押し付けがましい警告**: 「失敗情報」ばかり目立って、旅行自体が不安になるような見せ方は避ける（あくまで「知っていれば回避できる」というポジティブな注意喚起）。
