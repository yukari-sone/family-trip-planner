# 開発計画・ロードマップ

## 1. 目的
本ドキュメントは、FamilyTrip Planner（仮）を最短で検証し、価値を提供するまでの開発プロセス、タスク分解（WBS）、およびリスク管理方針を定義する。

## 2. 開発ロードマップ

### Phase 1: MVP開発（約1.5〜2ヶ月）
**ゴール**: 「他人のプランをコピーしてGoogleマイマップ(KML)で使う」価値と、「動画・しおりがもらえるなら振り返り入力をするか」の仮説検証。
- **実装対象**: 
  - アカウント登録（Supabase Auth）
  - プランの検索・閲覧・コピー機能
  - プランのドラッグ＆ドロップ編集機能
  - GoogleマイマップへのKMLエクスポート機能
  - 子供向け「旅のしおり」PDF生成機能
  - 振り返り入力（画像＋テキスト）と、BGM・アニメ付きムービー自動生成

### Phase 2: コミュニティ機能強化・UIブラッシュアップ（リリース後1〜2ヶ月）
**ゴール**: 投稿の質・量を向上させ、ユーザー間のエンゲージメントを高める。
- **実装対象**:
  - 「役に立った」アクション（いいね機能）
  - ユーザーランク（パスポート風のスタンプ付与機能）
  - ムービーのテーマ/BGMテンプレートの拡充

### Phase 3: 収益化・ネイティブ化の検討（リリース後半年以降）
**ゴール**: ビジネスとしての持続可能性を確保する。
- **実装対象**:
  - アプリのネイティブ化（PWAまたはReact Native等の検討）
  - プレミアム機能の課金基盤導入

## 3. タスク分解 (WBS) - Phase 1 MVP

MVP達成に向けた主要タスクを以下にブレイクダウンする。

### 1. 基盤構築・環境セットアップ
- [ ] Next.js (App Router) プロジェクトの初期構築
- [ ] Tailwind CSS / UIコンポーネントライブラリ(shadcn/ui等)のセットアップ
- [ ] Supabase プロジェクト作成 (DB, Auth, Storage)
- [ ] DBスキーマ定義とマイグレーションの実行
- [ ] Supabase RLS (Row Level Security) の設定

### 2. バックエンドAPI・コア処理実装
- [ ] KMLエクスポート処理（XML生成ロジックの実装）
- [ ] 旅のしおりPDF生成処理（html2canvas / react-pdf 等の検証・実装）
- [ ] ムービー自動生成基盤（Remotion等のセットアップとBGM・アニメテンプレートの実装）
- [ ] ムービー生成用API（非同期キューへの登録）の構築

### 3. フロントエンド実装
- [ ] **認証画面**: Google/SNSログイン・サインアップUIの実装
- [ ] **トップ・検索画面**: 出発ボード風デザイン、一覧・絞り込み機能の実装
- [ ] **プラン詳細画面**: タイムライン表示、アラートアイコン表示、コピー処理の実装
- [ ] **プラン編集画面**: `dnd-kit`を用いたドラッグ＆ドロップ並び替え機能の実装
- [ ] **マイページ**: パスポート風デザイン、所有プラン一覧の表示
- [ ] **振り返り入力画面**: スマホ最適化された画像アップロードと評価・タグ入力UI
- [ ] **ムービー待機・視聴画面**: 動画の再生プレイヤー、SNSシェアボタンの実装

### 4. テスト・リリース準備
- [ ] 初期シードデータ（サンプルプラン数十件）の作成とDB投入
- [ ] Vercelへのデプロイパイプライン構築
- [ ] スマートフォン実機でのUI/UXテスト
- [ ] 利用規約・プライバシーポリシーの作成

## 4. リスク管理

プロジェクト進行において想定されるリスクと対応策を事前に定義する。

| No | リスク内容 | 影響度 | 対策方針 |
|---|---|---|---|
| **R1** | **動画生成インフラの負荷増大・コスト超過** | 高 | Remotionのレンダリング処理を本番サーバーから分離（別Lambda等）し、タイムアウトや並列実行数を厳密に制御する。 |
| **R2** | **PDF/KML生成処理の技術的ハードル** | 中 | 開発初期の段階でPoC（概念実証）を行い、日本語フォントのレンダリング（PDF）やGoogleマイマップでの正常なインポート（KML）を検証する。 |
| **R3** | **初期コンテンツ不足によるユーザー離脱** | 高 | リリース前に、運営陣や友人（アンバサダー）による質の高いプランデータを数十件投入し、「検索すれば見つかる」状態を作っておく。 |
| **R4** | **プライベートデータの漏洩** | 致命的 | SupabaseのRLS設定を慎重に行い、他人の非公開データや画像にアクセスできないことをテストコードで自動検証する。 |
